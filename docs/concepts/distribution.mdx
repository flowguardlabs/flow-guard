FlowGuard supports several distribution patterns, but they do not all use the same product surface or contract path.

## Current Product View

| Pattern  | Current product model                                 | Best fit                                             |
| -------- | ----------------------------------------------------- | ---------------------------------------------------- |
| Airdrops | Claim-based campaign using `AirdropCovenant`          | open or restricted community distributions           |
| Grants   | Milestone-based budget plan                           | funded project work and contractor milestones        |
| Bounties | Single-stage budget-style release                     | one deliverable, one payout, one unlock condition    |
| Rewards  | Immediate or deferred single-recipient payout pattern | contributor bonuses, recognition, one-off incentives |

## Airdrops

Airdrops are the most explicit distribution product in the current app. They support:

- BCH or CashToken pools
- public campaigns
- restricted campaigns with Merkle proofs
- claim history
- pause and cancel controls

Use [Airdrops](/app/airdrops) when many recipients should claim from the same campaign.

## Grants

Grants are structured milestone releases. They work best when one recipient should receive funding over several stages.

Use [Grants](/app/grants) when:

- funding is committed up front
- release should happen in stages
- the recipient should only withdraw what has actually become due

## Bounties

Bounties are the simplest milestone pattern in the product today: one task, one amount, one release path.

Use [Bounties](/app/bounties) when:

- there is one deliverable
- the payout should stay locked until a time-based condition clears
- the recipient should receive the full amount in one release

## Rewards

Rewards cover one-off contributor payments and similar bonus-style distributions.

Use [Rewards](/app/rewards) when:

- the payout targets a specific wallet
- the amount is not part of a longer recurring schedule
- the distribution should be immediate or deferred, but still simple

## Contract Library vs Product Surface

The FlowGuard contract library includes lower-level distribution covenants such as:

- `AirdropCovenant`
- `GrantCovenant`
- `BountyCovenant`
- `RewardCovenant`

The current product does not expose each of those as an equally separate end-user surface. This documentation follows the current app experience first, while the contract reference pages describe the lower-level covenant set in detail.
