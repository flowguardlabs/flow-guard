title: 'Transfer Stream Recipient Rights'
description: 'How the current recipient transfers a transferable vesting stream to a new address.'
icon: 'arrow-right-arrow-left'
---

Transfer lets the current recipient reassign future claim rights for a transferable vesting stream.

## Supported Families

Transfer is supported for:

- `LINEAR`
- `STEP`
- `HYBRID`
- `TRANCHE`

Transfer is **not** supported for recurring streams.

<Note>The stream must be marked `transferable` and must still be `ACTIVE`.</Note>

## Flow

<Steps>
  <Step title="Open the stream">Navigate to the stream detail page.</Step>
  <Step title="Enter the new recipient">
    Use the transfer form to enter the new recipient's BCH P2PKH address.
  </Step>
  <Step title="Build the transfer transaction">
    The app calls `POST /api/streams/:id/transfer` with the current recipient as `x-user-address`.
  </Step>
  <Step title="Sign and broadcast">The current recipient signs the returned transaction.</Step>
  <Step title="Confirm the transfer">
    The app confirms the broadcast with `POST /api/streams/:id/confirm-transfer`, then refreshes the
    stream detail and activity feed.
  </Step>
</Steps>

## API Example

```json
POST /api/streams/:id/transfer
x-user-address: bchtest:qcurrent...

{
  "newRecipientAddress": "bchtest:qnew..."
}
```

## What Changes

- future claim rights move to the new recipient
- unclaimed vested value remains attached to the stream and becomes claimable by the new recipient after confirmation
- already claimed value stays settled to the old recipient
